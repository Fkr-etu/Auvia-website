# Règles développement Auvia

## Architecture

Toujours privilégier une architecture simple.

Ne jamais ajouter une dépendance sans justification.

---

## Code

Chaque feature doit avoir :

- tests
- documentation
- migrations si nécessaire

---

## Sécurité

Les documents utilisateurs sont privés.

Ne jamais exposer un document d'un autre utilisateur.

---

## IA

Toute réponse générée doit citer ses sources.

Ne jamais inventer une recommandation médicale.
